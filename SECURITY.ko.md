# 보안 정책

## 지원 버전

이 프로젝트는 초기 `0.1.0` 릴리스를 준비 중입니다. 안정적인 릴리스 프로세스가 확립되기 전까지 보안 수정은 기본 브랜치를 대상으로 합니다.

## 취약점 제보

`sec-issue-finder`에서 의심되는 취약점을 발견한 경우 공개 issue를 열지 말아 주세요.

가능하다면 저장소의 security advisory workflow를 사용해 주세요. 사용할 수 없다면 저장소 소유자가 명시한 보안 연락처를 통해 maintainer에게 비공개로 연락해 주세요.

포함하면 좋은 정보:

- 영향을 받는 버전 또는 commit
- 문제 설명
- 안전하게 공유할 수 있다면 재현 단계 또는 proof of concept
- 잠재적 영향
- 제안하는 완화 방법

## 보안 모델

`sec-issue-finder`는 lockfile을 신뢰할 수 없는 입력으로 읽고, 의존성 데이터를 정규화하며, 공개 보안 권고 소스를 질의하고, 알려진 보안 권고 매칭 결과를 보고합니다.

이 도구는 package manager 명령, 의존성 스크립트, auto-fix 로직을 실행하지 않습니다. 또한 완전한 취약점 커버리지를 주장하지 않습니다. 결과는 공개 보안 권고 데이터베이스와 사용 가능한 패키지/버전 메타데이터에 의존합니다.
